기획

어떤 문제를 왜 해결하려 했는가

논문 PDF를 다운로드하면 파일명이 abc_paper.pdf, download.pdf처럼 제목과 무관하게 저장되는 경우가 많습니다. 이 상태로 파일이 쌓이면 탐색기에서 논문을 다시 찾기 어렵고, 논문 제목을 일일이 열어서 확인해야 하는 불편이 생깁니다.

이 프로젝트는 이런 문제를 해결하기 위해 제작한 Windows 유틸리티 프로그램입니다. 사용자가 PDF를 드래그 앤 드롭하면 프로그램이 PDF 내부 정보를 읽고 제목을 추출한 뒤, 해당 논문 제목에 맞는 파일명으로 자동 변경하도록 구성했습니다.

구성물

주요 기능 목록
  • PDF 메타데이터 및 첫 페이지 텍스트 기반 제목 추출
  • 드래그 앤 드롭 방식 파일 추가
  • 선택 항목 제거, 전체 지우기, 일괄 파일명 변경
  • Windows 파일명에 사용할 수 없는 문자 자동 정리
  • 중복 파일명 발생 시 숫자를 붙여 안전하게 저장
  • 처리 결과를 로그로 출력하여 성공·실패 내역 확인

실현 방법

사용 기술, AI 도구 활용 과정

PythonPyMuPDFtkintertkinterdnd2

프로그램은 Python으로 구현하였고, GUI는 tkinter를 사용해 제작했습니다. PDF 제목 추출은 PyMuPDF를 이용해 메타데이터와 첫 페이지 텍스트를 분석하는 방식으로 구현했습니다. 또한 tkinterdnd2를 적용하여 사용자가 논문 파일을 프로그램 창에 바로 끌어다 놓을 수 있도록 만들었습니다.

제목 추출 과정에서는 메타데이터 제목이 적절한지 먼저 확인하고, 적절하지 않으면 첫 페이지에서 큰 글자 위주의 텍스트를 후보로 모아 제목을 추정하도록 구성했습니다. 이후 파일명으로 사용할 수 없는 문자를 제거하고, 동일한 파일명이 이미 존재하면 뒤에 번호를 붙여 충돌을 방지했습니다.

결과

완성된 프로그램 스크린샷 및 설명

아래 결과는 기존 abc_paper 파일을 프로그램에 드래그 앤 드롭 방식으로 넣은 뒤 파일명 변경 실행을 수행했을 때, 논문 제목에 맞게 파일 이름이 변경되는 과정을 보여줍니다. 실제 실행 결과에서 파일명이 논문 제목인 Three-Phase Coil Configurations for the Radial Suspension of a Bearingless Slice Motor.pdf로 변경되는 것을 확인할 수 있습니다.

변경 전: 탐색기에는 abc_paper로 저장되어 있고, 프로그램에는 해당 PDF가 추가된 상태입니다.
변경 후: 실행 결과 창과 처리 로그를 통해 파일명이 논문 제목 기준으로 정상 변경된 것을 확인할 수 있습니다.

특장점

이 프로그램만의 강점
  • 논문 관리에서 가장 불편한 파일명 정리 과정을 자동화할 수 있습니다.
  • 복잡한 설정 없이 드래그 앤 드롭만으로 사용할 수 있어 접근성이 좋습니다.
  • 논문 제목 기반 정리이므로 탐색기에서 논문 검색과 분류가 쉬워집니다.
  • 일반 사용자도 바로 실행 가능한 단순한 인터페이스로 구성했습니다.

한계점

현재 버전의 한계
  • 스캔 PDF처럼 텍스트 추출이 어려운 파일은 제목 인식 정확도가 낮을 수 있습니다.
  • 일부 논문은 메타데이터가 비어 있거나 첫 페이지 구성이 특이해 제목 추정이 실패할 수 있습니다.
  • 현재는 Windows 환경을 기준으로 제작되어 다른 운영체제 지원은 고려하지 않았습니다.
