PaperPilot
논문 PDF 제목 기반 파일명 변경 프로그램

다운로드한 논문 PDF의 파일명이 의미 없는 문자열인 경우가 많아, 탐색과 정리가 비효율적이라는 문제를 해결하기 위해 PDF 내부에서 제목을 추출하고 이를 기반으로 파일명을 자동 변경하는 데스크톱 프로그램을 구현하였다. 결과물은 사용자가 PDF를 드래그 앤 드롭하거나 추가 버튼으로 불러온 뒤, 예상 제목을 확인하고 한 번에 파일명을 바꾸는 방식으로 동작한다.

기획

어떤 문제를 왜 해결하려 했는가

논문을 웹에서 내려받으면 파일명이 논문 제목이 아니라 임의 문자열, 다운로드 번호, 시스템 생성 이름 등으로 저장되는 경우가 많습니다. 이 상태에서는 파일 탐색기에서 원하는 논문을 바로 찾기 어렵고, 폴더 정리와 재활용도 비효율적입니다.

따라서 사용자가 PDF를 직접 열지 않아도 파일명만 보고 논문을 식별할 수 있도록, PDF 내부의 제목을 자동으로 추출해 파일명에 반영하는 프로그램을 만드는 것을 목표로 했습니다.

문서 정리 시간 단축 가독성 높은 파일명 반복 작업 자동화 비전공자도 사용 가능

사용 시나리오

  • Google Scholar, IEEE, ScienceDirect 등에서 받은 PDF 파일명이 무의미한 경우
  • 여러 편의 논문을 한 폴더에 모아두고 빠르게 정리하고 싶은 경우
  • 연구실 자료, 과제 참고문헌, 개인 논문 아카이브를 체계적으로 관리하고 싶은 경우
  • 매번 수작업으로 제목을 복사해 파일명을 바꾸는 과정을 줄이고 싶은 경우

구성물

주요 기능 목록

PDF 추가 드래그 앤 드롭 또는 버튼 클릭으로 PDF 파일을 목록에 등록할 수 있습니다.
제목 추출 PDF 첫 페이지의 텍스트를 기반으로 논문 제목 후보를 자동 판별합니다.
목록 확인 현재 파일명과 예상 제목을 나란히 보여 주어 변경 전 검토가 가능합니다.
일괄 변경 여러 PDF를 한 번에 처리하여 파일명 변경 작업을 빠르게 수행합니다.
선택 제거 / 전체 지우기 잘못 넣은 항목을 정리할 수 있어 재작업이 쉽습니다.
처리 로그 성공/실패 내역을 하단 로그에 남겨 사용자가 결과를 확인할 수 있습니다.

실현 방법

사용 기술 및 AI 도구 활용 과정

1
문제 정의 논문 PDF 파일명이 제목과 다르게 저장되는 실제 불편을 기준으로 기능 요구사항을 정리했습니다.
2
GUI 설계 사용자가 쉽게 쓸 수 있도록 큰 드롭 영역, 목록 테이블, 실행 버튼, 로그 영역 중심으로 인터페이스를 구성했습니다.
3
핵심 로직 구현 PDF 텍스트 추출 후 제목 후보를 판별하고, 운영체제에서 허용 가능한 문자열로 정리한 뒤 파일명을 변경하도록 만들었습니다.
4
검증 및 보완 실제 PDF를 넣어 제목 추출과 파일명 변경 결과를 확인하고, 성공/실패 팝업과 로그를 추가해 사용성을 보강했습니다.
AI 도구는 아이디어 구체화, 예외 상황 점검, README/보고서 정리, UI 문구 다듬기 등에 활용하였다. 즉, 단순 코딩 보조를 넘어서 문제 정의 → 구현 → 문서화까지 이어지는 “바이브 코딩” 방식으로 프로젝트를 완성했다.

특장점

이 프로그램만의 강점

  • 문제 지향적입니다. 연구나 과제 중 실제로 자주 겪는 불편을 바로 해결합니다.
  • 사용 흐름이 단순합니다. 드래그 앤 드롭만으로 실행 가능해 별도 학습이 거의 필요 없습니다.
  • 반복 작업 자동화 효과가 큽니다. 여러 개 PDF를 수작업 없이 정리할 수 있습니다.
  • 결과가 즉시 눈에 보임니다. 파일 탐색기에서 변경된 파일명을 바로 확인할 수 있습니다.
  • 보고서 제출형 산출물까지 연결됩니다. 프로그램뿐 아니라 README, 패키징, HTML 보고서까지 확장 가능합니다.

왜 유용한가

이 프로그램은 거창한 대형 시스템이 아니라, 작지만 자주 반복되는 불편을 정확히 겨냥한 생산성 도구라는 점에서 가치가 있습니다. 특히 논문을 많이 다루는 학생, 연구실, 대학원 환경에서는 PDF 정리 자체가 시간이 많이 드는 작업인데, 이 과정을 자동화하면 탐색 시간과 피로도가 함께 줄어듭니다. 또한 GUI 기반으로 구현했기 때문에 코드에 익숙하지 않은 사용자도 접근할 수 있으며, 향후 메타데이터 기반 정리 도구나 연구 자료 관리 도구로 확장하기에도 적합합니다.

6

한계점

현재 버전의 한계

  • PDF마다 레이아웃이 달라 제목 판별이 항상 완벽하지는 않습니다.
  • 스캔본처럼 텍스트 추출이 어려운 PDF는 정확도가 떨어질 수 있습니다.
  • 영문 논문 중심의 제목 추출 규칙에 맞춰져 있어 일부 특수 형식 문서에서는 보완이 필요합니다.
  • 현재는 파일명 변경에 집중되어 있으며, 저자/학회/연도 기반 정리 기능은 포함되어 있지 않습니다.
  • 운영체제별 예외 처리, 파일명 길이 제한, 중복 파일명 처리 정책은 더 정교하게 다듬을 필요가 있습니다.

확장 가능성

향후에는 단순 파일명 변경 도구를 넘어, 논문 다운로드 후 자동 정리, 분야별 분류, 참고문헌 관리 프로그램과의 연계까지 확장할 수 있습니다. 즉, 본 프로젝트는 작은 자동화 도구로 시작했지만 연구 자료 관리 워크플로 전체를 개선하는 기반 기능으로 발전시킬 수 있습니다.

본 HTML 보고서는 단일 파일로 동작하며, 스크린샷 이미지를 base64로 내장하여 별도 이미지 파일 없이 제출 가능하도록 구성하였다.
